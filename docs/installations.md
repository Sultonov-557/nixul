# Installation Guide

This walkthrough shows how to install and operate Nixul on a fresh or existing NixOS machine. Follow it end-to-end for a reproducible setup.

## 1. Prerequisites
- A NixOS system (23.11 or newer recommended) with [flakes enabled](https://nixos.wiki/wiki/Flakes).
- Git available on the host.
- Optional but recommended: [`nh`](https://github.com/viperML/nh) for shorthand rebuild commands (`nix profile install nixpkgs#nh`).
- SSH access to the machine if you are provisioning remotely.

## 2. Bootstrap the repository
1. Switch to the system configuration directory (typically `/etc/nixos`).
2. Clone the repo:
   ```bash
   sudo rm -rf /etc/nixos
   sudo git clone <your-fork-url> /etc/nixos
   cd /etc/nixos
   ```
   > Replace `<your-fork-url>` with your fork so you can push host customizations.

## 3. Understand the layout
- `nix/hosts/<name>`: hardware profile plus the tags to load.
- `nix/modules/**`: feature modules grouped under `apps`, `core`, `desktop`, `dev`, `hardware`, and `services`.
- `nix/nixul/**`: shared options (users, aliases, keybinds, bookmarks, tags, themes).
- `docs/**`: architecture references, workflows, recovery procedures, and this installation guide.

## 4. Create your host
1. Copy an existing host as a starting point:
   ```bash
   cp -r nix/hosts/nomad nix/hosts/<your-hostname>
   ```
2. Edit `nix/hosts/<your-hostname>/default.nix`:
   - Set `networking.hostName` to your machine.
   - Import `hardware-configuration.nix` generated by `nixos-generate-config`.
   - Enable per-host toggles via `nixul.host.*` options if needed.
3. Review or merge the accompanying `bookmarks.nix` if you want host-specific browser bookmarks.

## 5. Pick tags and modules
Inside the host file you just created, locate the `loadTags` call:
```nix
loadTags [
  "desktop/hyprland"
  "dev/nixvim"
  "services/ollama"
]
```
- Remove tags you dont need.
- Add others from `nix/nixul/tags/**` to pull in desktops, services, per-user bundles, etc.
- For bespoke behavior create a small module under `nix/modules/**` and add a matching tag.

## 6. Validate and rebuild
Run the following from the repo root:
1. Format Nix files: `nix fmt`
2. Lint for dead code: `nix develop --command deadnix --fail .`
3. Evaluate all hosts: `nix flake check --all-systems --show-trace`
4. Switch the target host:
   ```bash
   nh os switch .#<your-hostname>
   # or
   sudo nixos-rebuild switch --flake .#<your-hostname>
   ```

## 7. Day-two workflows
- `nh os test .#<host>`: build and run activation tests without switching.
- `nh os build .#<host>`: perform builds on CI or headless environments.
- `nh os rollback .#<host>`: roll back to the previous generation if needed.
- Keep running `nix fmt` + `deadnix` before commits to stay consistent with CI.

## 8. Updating upstream
1. Add your fork as `origin` and the canonical repo as `upstream`.
2. Rebase regularly: `git fetch upstream && git rebase upstream/main`.
3. Push host changes to your fork. PRs should focus on generic modules or bug fixes so everyone benefits.

## 9. Troubleshooting tips
- If evaluation fails, run `nix develop` for a shell with every dependency used in CI.
- Secrets are managed via `sops-nix`; see `docs/secrets.md` to wire age/PGP keys.
- For recovery steps, reference `docs/recovery.md`.
- When in doubt, check `docs/architecture.md` for how the importer wires modules and tags.

Once you complete these steps, the machine will boot with the same curated experience showcased in the README. Happy rebuilding!
