# Hosts

## Structure

- Hosts live under `nix/hosts/<hostname>/`.
- A typical host directory contains:
  - `default.nix` – main host configuration.
  - `hardware-configuration.nix` – generated by `nixos-generate-config`.
  - optional extra files such as `bookmarks.nix` or host-specific modules.

## What goes in `default.nix`

The host file usually:

- imports `hardware-configuration.nix`;
- imports one or more users from `nix/users`;
- selects a theme via `loadTheme`;
- configures host-specific Home Manager settings (monitors, desktop tweaks);
- sets `nixul.host` attributes and the modules tree.

Example pattern:

```nix
nixul.host = {
  name = "nomad";
  timezone = "Asia/Tashkent";
  location = "Tashkent"; # optional
  bookmarks = import ./bookmarks.nix;

  modules =
    lib.recursiveUpdate
      (loadTags [
        "base"
        "wayland"
        "desktop"
        # ... other tags
      ])
      {
        core.maintenance.auto-upgrade.enable = true;
        services.ai.ollama.enable = true;
      };
};
```

- `loadTags` returns an attribute set mirroring the module tree.
- Inline overrides are merged on top with `lib.recursiveUpdate`.

## Using tags

- Tags are defined under `nix/nixul/tags` and act as reusable profiles.
- Common examples include `base`, `desktop`, `dev/*`, `gaming/*`, `ai/*`.
- Changing the tags list for a host is often enough to adjust its role (for example adding `gaming/steam` or `dev/docker`).

## Adding a new host

1. Copy an existing host directory:
   ```sh
   cp -r nix/hosts/nomad nix/hosts/<new-host>
   ```
2. Regenerate hardware config on the new machine:
   ```sh
   nixos-generate-config --root /mnt
   cp /mnt/etc/nixos/hardware-configuration.nix \
      nix/hosts/<new-host>/hardware-configuration.nix
   ```
3. Edit `nix/hosts/<new-host>/default.nix`:
   - update `nixul.host.name`, `timezone`, and `location`;
   - adjust `loadTags` to match the machines role;
   - tweak `nixul.host.modules` overrides.
4. Ensure your user is imported from `nix/users`.
5. Build and switch on the target machine:
   ```sh
   nh os switch .#<new-host>
   ```

## Host-specific overrides

Use `nixul.host.modules` for host-only differences:

- enable a service only on a desktop host;
- turn on additional dev tools on work machines;
- disable heavy dashboards on laptops.

If an override becomes generally useful, move it into a shared tag or module so other hosts can reuse it.
